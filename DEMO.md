# ✦ Afterlight — Hackathon Demo Script

> **3-minute demo guide** for the Gemma 4 Good Hackathon submission.
> Follow this sequence exactly. Timestamps are targets, not hard limits.

---

## Before You Record — Pre-Demo Checklist

Do **all of this the day before**, not 5 minutes before recording.

### ✅ Infrastructure
- [ ] Ollama is running (`ollama list` shows `gemma4:e2b`)
- [ ] Backend is running on port 8000 (`/api/health` returns `ollama.connected: true`)
- [ ] Frontend is running on port 3000 (landing page loads)
- [ ] Browser is Chrome or Edge, zoom level 100%, resolution 1920×1080

### ✅ Pre-create the Memory Space (do this before recording)
You do NOT want to type during the demo — create the space in advance.

1. Go to http://localhost:3000/create
2. Fill in:
   - **Name**: `Nani` (or your sample persona name)
   - **Relationship**: Grandmother
   - **Birth year**: 1942
   - **Passed year**: 2023
   - **Language**: Hindi, English (Hinglish)
   - **Description**: A warm, wise woman who told stories in a mix of Hindi and English, loved cooking, and always had advice ready.
3. Click **Continue to Agreement** → check all 6 boxes → Submit
4. Note the space ID from the URL: `/spaces/SPACE_ID/...`

### ✅ Pre-upload and pre-process sample data
The sample text files are in `sample_data/`. Upload them via the API or the UI before recording.

**Fastest way — via curl:**
```bash
# Upload English sample
curl -X POST "http://localhost:8000/api/memory-spaces/YOUR_SPACE_ID/assets" \
  -F "file=@sample_data/sample_story_en.txt"

# Upload Hindi/Hinglish sample
curl -X POST "http://localhost:8000/api/memory-spaces/YOUR_SPACE_ID/assets" \
  -F "file=@sample_data/sample_story_hi.txt"
```

**Then trigger processing:**
```bash
curl -X POST "http://localhost:8000/api/memory-spaces/YOUR_SPACE_ID/process"
```

Wait until processing completes (check status):
```bash
curl "http://localhost:8000/api/memory-spaces/YOUR_SPACE_ID/setup-status"
```

When all steps show `done` — you're ready.

### ✅ Pre-generate training data
```bash
curl -X POST "http://localhost:8000/api/memory-spaces/YOUR_SPACE_ID/generate-training-data"
```

### ✅ Pre-train the adapter (if you have a GPU)
Run this well in advance (can take hours on first run):
```bash
# Get the job_id from the training-data step, then:
curl -X POST "http://localhost:8000/api/memory-spaces/YOUR_SPACE_ID/train-adapter"
```

If training completed, the Setup page will show **"Adapter Ready ✓"** — this is what you want on screen.

If you don't have a GPU, the Setup page will honestly show **"Training requires CUDA GPU"** — that's fine, just explain it verbally.

### ✅ Pre-warm the Talk page
Navigate to the Talk page for your space, send one message, confirm it responds. This caches the persona in memory so the first on-screen message is fast.

## The 3-Minute Demo Script

### ⏱️ 0:00–0:20 — Hook: What is Afterlight?

**Show:** Landing page at `localhost:3000`

**Say:**
> "This is Afterlight — a private, local-first presence engine built on Gemma 4.
> It preserves the voice, memories, and wisdom of a loved one — so future generations can truly know them.
> Everything runs on your machine. No cloud. No data leaving your home."

**Action:** Slowly scroll down the landing page — hero, pillars (Stories, Voice, Wisdom), how it works.

---

### ⏱️ 0:20–0:40 — Setup Transparency

**Show:** Navigate to `localhost:3000/spaces/YOUR_SPACE_ID/setup`

**Say:**
> "The Setup page is built on radical honesty. It shows you exactly what tools are available, what the pipeline has done, and what's missing — in plain language.
> There is no fake AI. If Ollama isn't running, the app says so."

**Action:** Show the pipeline steps (all green checkmarks if pre-processed), the tool status section, and the adapter status.

**Point to the adapter status specifically:**
> "The adapter was trained beforehand using QLoRA fine-tuning on `google/gemma-4-e2b-it` — the real Gemma 4 model — using synthetic training data generated by Gemma 4 itself from the extracted memories."

---

### ⏱️ 0:40–1:10 — Memory Capture

**Show:** Navigate to `localhost:3000/spaces/YOUR_SPACE_ID/capture`

**Say:**
> "The Capture Studio accepts audio, video, images, documents, and text.
> Everything is processed locally — faster-whisper for transcription, Tesseract for OCR, FFmpeg for video.
> The files are already uploaded here — you can see them."

**Action:** Show the uploaded assets list. Then point to where you would trigger processing (or show it's already done).

---

### ⏱️ 1:10–1:45 — Memory Cards & Review

**Show:** Navigate to `localhost:3000/spaces/YOUR_SPACE_ID/review`

**Say:**
> "Gemma 4 reads every evidence chunk and extracts structured memory cards — each with a title, summary, source quote, themes, and values.
> As a steward, you review and approve memories before they are used. Flagged memories are never used in conversations."

**Action:** Show 2-3 memory cards. Click one to show the detail. Show the Approve / Flag buttons.

---

### ⏱️ 1:45–2:30 — The Conversation (Main Feature)

**Show:** Navigate to `localhost:3000/spaces/YOUR_SPACE_ID/talk`

**Say:**
> "This is where Afterlight comes alive. You're talking with a presence built from real memories — not a generic chatbot.
> Every response is grounded in what we actually know. Gemma 4 retrieves relevant memories from ChromaDB via RAG and generates a first-person reply in their voice."

**Action 1:** Type the first question (use one of these):
- `"What mattered most to you in life?"`
- `"Nani, what did you always say about difficult times?"`
- `"Tell me something about your childhood."`

Wait for the response (3–10 seconds depending on GPU).

**Point to Trust Chip:**
> "See this chip — 'Memory-backed'. That means this response is directly grounded in approved memory cards. You can click Sources to see exactly which memories were used."

**Action 2:** Click the Sources drawer — show the memory card citations.

**Action 3 (optional):** Ask a boundary-crossing question:
- `"Are you really Nani? Are you conscious?"`

**Say:**
> "Watch what happens when you cross the line. The system doesn't pretend — it's transparent about what it is."

Show the `system-boundary` trust chip response.

---

### ⏱️ 2:30–2:50 — Timeline (Bonus Feature)

**Show:** Navigate to `localhost:3000/spaces/YOUR_SPACE_ID/timeline`

**Say:**
> "Afterlight also builds a life timeline from the extracted memories — events, places, years — all automatically structured by Gemma 4 from the uploaded content."

---

### ⏱️ 2:50–3:00 — Close

**Show:** Back to landing page or API docs

**Say:**
> "Afterlight is local-first, honest, and evidence-backed.
> It runs on Gemma 4 — the most capable model for this task — fully on-device.
> The full source code, training script, generated dataset, and adapter weights are all in the repo.
> Thank you."

## Things to Say if Something Goes Wrong

| Problem | What to say |
|---|---|
| Response is slow | "Gemma 4 is generating locally on CPU — on a GPU this takes under 2 seconds." |
| Response doesn't come | Refresh the Talk page. Re-send. If still broken — check Ollama is running. |
| "Ollama not connected" appears | "This shows the honest error state — Ollama needs to be running. Let me start it." Run `ollama serve`. |
| Memory cards are empty | "Processing is triggered asynchronously — let me show the setup page where we can see the pipeline ran already." Navigate to Setup page. |
| Adapter shows "not trained" | "Training requires a CUDA GPU which we're running on CPU today — but the full training script and JSONL dataset are in the repository and ran successfully on our training machine." |

## Demo Recording Tips

1. **Use OBS Studio** — free screen recorder. Set to 1920×1080, 30fps.
2. **Record in one take** if possible — judges can tell when it's spliced.
3. **Keep browser zoom at 100%** — UI is designed for this.
4. **Turn off notifications** — Windows Focus Assist → Priority Only.
5. **Close Slack, Discord, email** before recording.
6. **Use a wired internet connection** for Ollama API calls (even local, it matters for stability).
7. **Have a backup screen recording** — record twice, use the better one.
8. **Edit out long pauses** — Gemma 4 thinking for 8 seconds is fine; 30 seconds is not.
